# A股 1进2 连板接力 UZI 智能评委席批量审计设计方案

* **日期**：2026-06-24
* **版本**：1.0
* **作者**：Antigravity (Oh My Pi Coding Agent)
* **状态**：设计中 (待评审)

---

## 1. 背景与目标 (Background & Goals)

目前控制台已完成了 1进2 首板股的量化筛选与模拟账本。然而，短线接力选手依然面临**基本面地雷**（如商誉减值、应收账款爆雷、大股东高质押）与**多流派博弈态度**（价值派是否认可、游资派是否合力）的辨识难点。

本项目旨在将 **wbh604/UZI-Skill 智能评委席深度审计能力** 批量化、一键式引入看板中：
* **量化与质检闭环**：每日复盘筛选出 Top 5 重点个股后，系统自动在后台以混合架构模型执行 UZI 评测。
* **大模型与规则混合 (Hybrid Execution)**：首选调用本地 `UZI-Skill` 子进程，利用 Claude/Gemini 等大模型评审团进行诊断；若无 API 密钥，自动降级执行本地财务规则模拟器，保证系统的高可用性。
* **数据展现**：在看板正中新增 `UZI 智能评委席报告` 选项卡，展示综合评分、评委投票分布（价值/游资/排雷）及诊断摘要，并提供一键调阅完整 UZI HTML 报告的超链接。

## 2. 大模型子进程管线与报告解析 (Subprocess & HTML Parser)

当系统检测到 API 密钥（如 `GEMINI_API_KEY`）存在时，执行大模型子进程调用管线：

1. **项目自检测与自动克隆 (Auto-Clone)**：
   * 检查 `../UZI-Skill/run.py` 是否存在。若缺失，自动运行子进程：
     `git clone https://github.com/wbh604/UZI-Skill.git ../UZI-Skill`
   * 若克隆因网络超时等原因失败，系统优雅降级为执行第 3 节的离线财务规则模拟。
2. **大模型诊断批处理 (Batch Run)**：
   * 循环对今日 Top 5 重点标的，在后台串行/并行调阅 UZI-Skill 快速扫描指令（不打开浏览器）：
     `python ../UZI-Skill/run.py {代码} --depth lite --no-browser`
3. **HTML 报告解析与入库 (HTML Parser)**：
   * 诊断完成后，读取生成的 `../UZI-Skill/reports/{date}_{code}.html` 报告。
   * 使用正则表达式提取以下关键段落与评分：
     * **综合分值**（例如：`Jury Score: 85` 或 `最终评议: 82分`）。
     * **评委表决**：巴菲特（价值）多空态度、赵老哥（游资）多空态度。
     * **排雷等级**：Michael Burry（大空头）排雷级别。
     * **核心评语**：提取评审总结中的核心文字段落。
   * 解析完成后将数据存入 SQLite 数据库 `uzi_audit` 历史表中。

## 3. 本地财务规则模拟器 (Local Rule-based UZI Emulator)

当系统处于离线状态（缺少 API 密钥或项目未完成克隆）时，自动降级运行本地财务规则模拟器，利用 `mootdx` 季报财务及 F10 数据，模拟 UZI 的三个核心评委席位：

1. **巴菲特席位 (Value Seat - 价值评委)**：
   * **净资产收益率 (ROE)**：$ROE \ge 15\%$ 得 40 分；$ROE \ge 8\%$ 得 20 分；$ROE < 0$ 扣 20 分。
   * **资产负债率 (Debt Ratio)**：负债率 $< 45\%$ 得 30 分；$> 70\%$ 扣 15 分。
   * **每股收益 (EPS)**：$EPS \ge 0.5$ 元得 30 分；$EPS < 0$ 扣 15 分。
   * 最终折合为 $0 \sim 100$ 的价值流派得分与态度：$\ge 80$ 为多头，$< 50$ 为空头，其余为观望。
2. **赵老哥席位 (Momentum Seat - 游资评委)**：
   * **板块共振**：同行业首板涨停数 $\ge 5$ 家得 40 分；$\ge 3$ 家得 20 分。
   * **首次封板时间**：$\le 09:35:00$ 得 30 分；$\le 10:30:00$ 得 15 分；下午板扣 10 分。
   * **换手率区间**：换手率处于 $4\% \sim 12\%$ 黄金换手区间得 30 分。
   * 最终折合为 $0 \sim 100$ 的情绪接力得分与态度：$\ge 85$ 为多头，$< 55$ 为空头，其余为观望。
3. **大空头席位 (Short/Trap Seat - 排雷评委)**：
   * **应收账款比率**：应收账款占比总资产 $> 35\%$，判定为“高风险回款存疑”，扣 30 分。
   * **商誉减值隐患**：商誉占比净资产 $> 20\%$，判定为“商誉爆雷隐患”，扣 20 分。
   * **ST/退市排查**：简称含 ST 或退市直接标记为“高危”。
   * 输出排雷等级（安全 / 关注 / 高危 / 极度危险）。
4. **决策融合与建议生成**：
   * 三个席位得分求均值作为模拟综合评分。使用格式化模板生成段落式建议，存入 `uzi_audit` 数据库。

## 4. 数据库升级与数据导出 (Database & Export)

1. **`uzi_audit` 历史数据表**：
   ```sql
   CREATE TABLE IF NOT EXISTS uzi_audit (
       date TEXT,
       code TEXT,
       name TEXT,
       average_score REAL,
       val_vote TEXT,
       mom_vote TEXT,
       risk_level TEXT,
       summary TEXT,
       report_path TEXT,
       PRIMARY KEY (date, code)
   )
   ```
2. **数据导出 (`export_data()`)**：
   * 每次生成看板数据时，查询 `uzi_audit` 并作为 `window.RECAP_UZI_AUDIT` 导出到 `recap_history.js`。

## 5. 前端展示与展现设计 (Frontend UI & Integration)

1. **主看板新增全宽 Bento 卡片**：
   * 标题：`■ UZI 智能评委席报告 / UZI JURY AUDIT REPORT`
   * 右侧状态标签：`[大模型评审模式]`（绿色边框/背景）或 `[本地财务规则模拟]`（黄色边框/背景）。
2. **个股多维度评审展示 (Top 5 Jury Grid)**：
   * **综合评分**：大字等宽显示（如 `85` 分），高分标红，低分标绿/灰。
   * **评委席表决状态 (Jury Votes)**：
     * 价值流派 (巴菲特席位)：显示 `多头 / 空头 / 观望`（红色多，绿色空，灰色观望）。
     * 游资流派 (赵老哥席位)：显示 `多头 / 空头 / 观望`。
     * 排雷流派 (大空头席位)：显示 `安全 / 关注 / 高危 / 极度危险`（高亮提示）。
   * **核心评语摘要 (Jury Summary)**：以微型字体（`text-3xs`）展示解析出的核心诊断意见，提供“查看 UZI 深度诊断报告”链接。

## 6. 验证方案 (Test & Validation Plan)

* **自适应降级测试**：拔掉网络线或清空 API 密钥，执行复盘。验证控制台成功降级为本地财务模拟打分，无数据库插入报错，且前端展示 `[本地财务规则模拟]` 状态。
* **大模型运行测试**：在配置有效 `GEMINI_API_KEY` 的环境下，验证成功在后台调用 UZI 子进程并顺利生成/解析 HTML 报告。
* **UI 对齐与无障碍审计**：使用 Chromium 进行页面 console 日志诊断，确认 `index.html` 正常挂载。
