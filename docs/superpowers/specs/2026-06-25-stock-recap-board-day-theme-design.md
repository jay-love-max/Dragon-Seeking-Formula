# stock-recap-board 日间主题 / 双主题设计方案

* **日期**：2026-06-25
* **版本**：1.0
* **作者**：Antigravity (Oh My Pi Coding Agent)
* **状态**：设计中（待评审）
* **结论**：采用 **A. 米白纸感双主题** 作为日间主题方向，并保留现有深色主题。

---

## 1. 背景与目标

当前看板整体是深色金融终端风格：适合夜间盯盘，但白天长时间复盘时偏暗、偏重、容易疲劳。

本次设计目标：

1. **新增日间主题**：提供一套浅色视觉系统，白天默认更舒适。
2. **保留深色主题**：不破坏现有夜间高对比、强情绪的浏览体验。
3. **统一主题入口**：主题切换应是同一套状态系统，而不是两份独立页面。
4. **尽量少动数据逻辑**：只改视觉层、交互层和少量主题感知配置，不碰复盘计算与数据导出链路。

---

## 2. 主题方向

### 2.1 默认日间主题：米白纸感

采用接近报表 / 纸张 / 早盘复盘的浅色调：

- 页面背景：米白、暖灰、浅纸色
- 卡片底：更浅的白色或暖白
- 边框：低对比细线
- 阴影：更轻、更软，避免漂浮感
- 强调色：沿用当前红色系作为主强调，不扩展过多新颜色

### 2.2 深色主题：继续保留

深色主题不重做，只做必要的 token 抽象，以便与日间主题共享结构。

深色主题继续承担：

- 夜盘 / 盘后复盘
- 高密度信息展示
- 更强的情绪提示与警示

---

## 3. 设计原则

1. **信息优先**：主题切换不改变信息结构，只改变视觉表达。
2. **克制改色**：只做背景、边框、文字、卡片、图表等必要的主题化，不引入第二套风格语言。
3. **单一强调色**：红色继续作为核心动作与风险强调色，减少多色竞争。
4. **可读性优先**：白天主题以长时间阅读舒适度为第一目标。
5. **可回退**：若主题变量缺失或系统不支持，默认仍能正确显示现有深色样式。

---

## 4. 组件级主题映射

### 4.1 页面背景与全局文字

- 日间背景：米白 / 浅暖灰渐变
- 全局文字：深灰而非纯黑，降低视觉压迫
- 次级文字：中灰，保证层级清晰

### 4.2 卡片与容器

- 卡片底色更轻，透明度更低
- 卡片边框使用细线而非重边
- 阴影收敛，避免暗色主题那种厚重浮层感
- 保留当前卡片的“金融控制台”秩序感

### 4.3 顶部栏与控制区

- 顶部栏改为浅底 + 细边线
- 日期选择、时段模拟、搜索等控件使用浅底输入框
- 控件 hover / focus 继续使用红色强调，但强度减弱

### 4.4 表格、指标、徽标

- 表格分隔线更淡，减少黑色网格感
- 数值标签保留高亮，但避免过饱和背景
- 重要状态仍使用红 / 黄 / 绿作语义提示

### 4.5 图表

Chart.js 图表需要感知当前主题：

- 深色主题：沿用现有图表配色
- 日间主题：更新坐标轴、网格线、图例与线条的颜色，使其在浅底上保持清晰

图表不能只换背景不换坐标颜色，否则会显得“半夜版贴到白底上”。

---

## 5. 交互与状态

### 5.1 切换方式

提供一个主题开关：

- 跟随系统
- 日间主题
- 夜间主题

### 5.2 默认策略

建议默认顺序：

1. 若用户有手动选择，优先使用手动选择
2. 否则跟随系统 `prefers-color-scheme`
3. 若系统无法判断，默认深色（保持当前行为安全）

### 5.3 持久化

用户主题选择应保存在本地存储中，刷新后保持一致。

---

## 6. 实现边界

### 6.1 仅修改前端

本次主题工作限定在前端视觉层：

- `index.html` 的样式变量
- Vue 主题状态
- 图表主题感知
- 本地存储

### 6.2 不修改的数据链路

以下内容不在范围内：

- `src/recap_engine.py` 的复盘逻辑
- SQLite 结构变更
- 交易日、题材、评分、晋级率计算逻辑
- 数据导出格式

---

## 7. 验证方案

1. **视觉切换验证**：浅色 / 深色主题切换后，页面无布局错位，文字和控件可读。
2. **状态持久化验证**：刷新页面后，主题保持上次选择。
3. **系统跟随验证**：在系统浅色 / 深色偏好下，默认主题选择符合预期。
4. **图表验证**：切换到日间主题后，图表坐标、网格、图例均适配浅底。
5. **回退验证**：关闭主题逻辑时，深色主题仍可完整使用。

---

## 8. 交付标准

满足以下条件即可认为本设计完成：

- 现有看板拥有可切换的日间 / 夜间双主题
- 日间主题默认采用米白纸感风格
- 视觉层与数据层解耦
- 图表与主要组件在两种主题下均保持可读
- 主题选择可持久化
