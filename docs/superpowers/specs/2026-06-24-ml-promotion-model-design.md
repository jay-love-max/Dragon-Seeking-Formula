## 1. 背景与目标 (Background & Goals)

目前 A股 1进2 连板接力控制台的“接力指数”打分基于静态的人工规则。虽然这些规则凝聚了短线筹码的核心逻辑，但无法动态评估大盘环境、行业共鸣及特征间的协同效应。

本项目旨在引入 **scikit-learn 随机森林分类器 (Random Forest Classifier)**，实现每日盘后动态在线训练：
* **选股与预测**：每次运行复盘时，系统自动提取历史数据进行就地训练，并输出明日 1 板候选股的真实晋级成功概率（0% - 100%）。
* **双轨并行展示**：在前端保留经验规则“接力指数”的同时，并排展示量化预测的“预估晋级率”，实现多维交叉验证。
* **零配置与零部署**：动态训练在 0.2 秒内完成，免去外部大模型及离线文件的维护负担。

## 2. 特征工程 (Feature Engineering)

输入模型（特征矩阵 $X$）的变量包括个股筹码特征、行业板块共振及大盘情绪特征，共 10 维自变量：

### 2.1 个股筹码特征
1. **首次封板时间秒数 (`first_seal_time_seconds`)**：将 `首次封板时间`（如 `093500`）转换为距离开盘（`09:25:00`）的累计秒数。
   * 计算公式：$\text{hour} \times 3600 + \text{minute} \times 60 + \text{second} - 33900$
   * 例：一字板 `092500` $\rightarrow 0$，秒板 `093100` $\rightarrow 360$。
2. **日内炸板次数 (`blown_count`)**：直接使用原始整数值。
3. **日内换手率 (`turnover`)**：直接使用原始浮点值（百分比）。
4. **流通市值 (`float_mcap`)**：直接使用原始浮点值（单位：亿元）。
5. **封单比 (`seal_ratio`)**：
   * 计算公式：$\text{封板资金} / \text{流通市值} \times 100\%$。

### 2.2 行业板块特征
6. **行业目标编码 (`sector_target_enc`)**：
   * 对股票所属的“行业板块”（如半导体、元件）采用目标编码（Target Encoding）：计算该行业在历史首板股中成功晋级为二板的均值概率。
   * **冷启动与边缘行业**：如果该行业在历史数据中未曾出现过，则填充为**全局历史平均晋级率**（如 `0.12`），防止出现空值导致模型报错。

### 2.3 大盘情绪特征
7. **大盘涨停数 (`market_limit_ups`)**：当日全市场封死涨停的个股家数。
8. **大盘跌停数 (`market_limit_downs`)**：当日全市场封死跌停的个股家数（代表恐慌度）。
9. **大盘两市成交额 (`market_turnover`)**：当日上证与深成指成交额的总计（单位：十亿元）。
10. **大盘晋级胜率 (`market_promo_rate`)**：当日昨日首板在今天的晋级率（代表短线接力溢价）。

### 2.4 模型标签定义 (Target $Y$)
* **目标分类 $Y \in \{0, 1\}$**：
  * 对于日期 $T$ 的首板样本（`consecutive_boards == 1`），如果该股票在 $T+1$ 交易日的 `limit_ups_archive` 存档中且 `consecutive_boards == 2`（晋级成功），则标记为 $Y=1$；否则为 $Y=0$。

## 3. 模型训练管线 (Model Pipeline & Training)

### 3.1 动态训练原则与数据防泄露 (No Look-Ahead Bias)
* **动态训练**：每次执行复盘时，系统自动连接 SQLite，只读取**交易日期 $< T$ （过去的历史）**的数据作为训练集。
* **泄露防范**：严禁将 $T$ 日或以后的数据混入训练集中，确保历史回测概率与实战预测概率完全一致。

### 3.2 随机森林超参数配置 (Hyperparameters)
* **算法选择**：`sklearn.ensemble.RandomForestClassifier`
* **参数配置**：
  * `n_estimators=150`：集成150棵树，保证袋外估计（OOB）和概率预测的平滑稳定性。
  * `max_depth=6`：限制最大树深，防止在千条样本级别的短线数据集上过拟合。
  * `min_samples_leaf=2`：每个叶子节点最少需要2个样本，提高泛化能力。
  * `random_state=42`：固定随机种子，确保回测结果可重现。

### 3.3 特征重要性打印 (Feature Importance Log)
训练完成后，在终端输出特征重要性（Feature Importances）排名，提供量化策略透明度：
```text
[量化模型] 随机森林动态训练成功 // 历史训练样本数: 1240 组
[量化模型] 特征重要性权重排名 (Feature Importances):
  1. 封单比 (seal_ratio): 31.4%
  2. 日内换手率 (turnover): 22.8%
  3. 行业历史晋级率 (sector_target_enc): 18.5%
  4. 两市成交额 (total_turnover): 14.2%
  5. 首次封板秒数 (first_seal_time_seconds): 13.1%
```

### 3.4 冷启动保护 (Cold Start)
* 如果训练集历史样本数 $< 30$ 组，模型将不执行 `fit` 过程，默认输出预测概率为 `None`。前端看板会自动检测并静默隐藏概率标签。

## 4. 数据库升级方案 (Database Migration)

为了存储预测出的晋级概率，且不破坏用户已有的 SQLite 历史数据：
* **自适应检测升级**：在 `init_db()` 中，读取 `candidates` 的表结构。如果 `pred_prob`（预测概率）字段不存在，则通过 `ALTER TABLE candidates ADD COLUMN pred_prob REAL` 进行无损原地升级。

## 5. 前端展示与展现设计 (Frontend UI & Integration)

1. **数据导出 (`export_data()`)**：
   * 从数据库中读取 `pred_prob` 并映射到 JSON 属性。导出时乘以 100 以便前端直观展示（如 `0.214` 导出为 `21.4`）。
2. **前五强个股卡片 (Bento Cards)**：
   * 原有 `接力指数: 120` 右侧增加一个新的红框 Badge：`预估晋级率: 21.4%`。
3. **首板候选股票池表格 (Table View)**：
   * 在 `接力指数` 列左侧，新增一列 `预估晋级率`。数据缺失时显示 `N/A`。
4. **决策模拟器联动 (Simulator Linkage)**：
   * 用户点击个股卡片聚焦决策器时，顶部会自动显示该股的量化预估晋级率。

## 6. 验证方案 (Test & Validation Plan)

* **单元测试**：编写测试函数，验证封板时间转换函数能准确转换秒数。
* **数据流测试**：检查在预测日期 $T$ 时，训练集的最大日期确实小于 $T$（无未来数据泄露）。
* **冷启动测试**：人为限制数据库只有 10 条样本，验证系统能正常打印警告并降级，前端网页显示 `N/A`，无任何 JS 报错。
